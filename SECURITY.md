# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | Yes                |

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability within this project, please follow these steps:

### How to Report

1. **Do not** open a public GitHub issue for security vulnerabilities
2. Send an email to **support@next8n.com** with details about the vulnerability
3. Include the following information:
   - Description of the vulnerability
   - Steps to reproduce the issue
   - Potential impact
   - Any suggested fixes (optional)

### What to Expect

- **Acknowledgment:** We will acknowledge receipt of your report within 48 hours
- **Investigation:** We will investigate the issue and determine its impact
- **Resolution:** We will work on a fix and coordinate disclosure timing with you
- **Credit:** We will credit you in the release notes (unless you prefer to remain anonymous)

### Scope

This security policy applies to:
- All code in this repository
- Documentation and templates
- Configuration files

### Best Practices

When using this framework:

1. Keep all dependencies up to date
2. Use environment variables for sensitive data
3. Follow the principle of least privilege
4. Review code changes before deployment
5. Enable logging and monitoring in production

## Contact

For security concerns, contact: **support@next8n.com**

For general questions, please use GitHub Issues.
